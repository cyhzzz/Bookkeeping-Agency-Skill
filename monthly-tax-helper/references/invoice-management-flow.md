# 票据录入管理流程

## 票据状态机

### 状态定义

| 状态代码 | 状态名称 | 说明 |
|----------|----------|------|
| PENDING | 待识别 | 票据已上传，等待OCR识别 |
| RECOGNIZING | 识别中 | OCR正在处理 |
| RECOGNIZED | 已识别待确认 | OCR识别完成，需人工确认 |
| CONFIRMED | 已确认 | 人工确认票据信息无误 |
| VOUCHERED | 已生成凭证 | 已生成对应记账凭证 |
| ARCHIVED | 已归档 | 月末结账后归档 |

### 状态流转图

```
[PENDING] → [RECOGNIZING] → [RECOGNIZED] → [CONFIRMED] → [VOUCHERED] → [ARCHIVED]
   ↑                                                                             │
   └──────────────────────────── 月末结账后批量更新 ─────────────────────────────┘
```

### 状态触发条件

| 当前状态 | 触发事件 | 下一个状态 | 触发条件 |
|----------|----------|------------|----------|
| PENDING | 上传完成 | RECOGNIZING | 票据文件成功上传 |
| RECOGNIZING | OCR完成 | RECOGNIZED | 识别置信度≥95% |
| RECOGNIZING | OCR完成 | RECOGNIZED | 识别置信度<95%，需人工确认 |
| RECOGNIZED | 人工确认 | CONFIRMED | 用户确认所有字段 |
| CONFIRMED | 生成凭证 | VOUCHERED | 凭证生成成功 |
| VOUCHERED | 月末结账 | ARCHIVED | 会计期间结账完成 |
| RECOGNIZED | 人工修正 | CONFIRMED | 用户修正并确认 |

### 异常状态

| 状态代码 | 状态名称 | 说明 |
|----------|----------|------|
| SUSPICIOUS | 疑似假票 | 比对税务局数据库失败 |
| OVERDUE | 逾期票据 | 开票日期超过90天 |
| AMOUNT_ABNORMAL | 金额异常 | 单张金额超10万或异常低 |
| DUPLICATE | 疑似重复 | 发票号码重复 |

## 票据录入标准操作流程（SOP）

### Step 1: 票据接收

#### 1.1 纸质票据接收

**接收标准**：
- 拍照要求：光线均匀、无反光、字迹清晰
- 分辨率要求：≥300DPI
- 拍摄角度：垂直于票据平面
- 文件格式：JPG/PNG
- 每张票据单独保存，避免折叠

**不合格票据处理**：
- 光线过暗/过亮 → 要求重新拍摄
- 存在反光 → 调整角度重新拍摄
- 字迹模糊 → 要求提供清晰版本

#### 1.2 电子票据接收

**接收格式**：
- PDF格式（首选）
- OFD格式（国产标准）
- XML格式（电子发票源文件）
- 图片格式（JPG/PNG，仅限电子发票截图）

**文件命名规范**：
```
{客户编号}_{票据类型}_{开票日期}_{发票号码}.{后缀}
示例：KH001_专票_20240115_12345678.pdf
```

#### 1.3 收集时间要求

| 客户类型 | 提交截止日 | 说明 |
|----------|------------|------|
| 一般纳税人 | 次月5日前 | 需抵扣进项税 |
| 小规模纳税人 | 次月10日前 | 季度申报 |
| 年度汇算清缴 | 次年5月31日前 | 企业所得税 |

---

### Step 2: 票据上传

#### 2.1 上传前准备

**排序要求**：
- 按开票日期顺序排列
- 同一日期内按发票号码顺序
- 收入类票据在前，支出类票据在后

**分类要求**：
| 票据类型 | 代码 | 说明 |
|----------|------|------|
| 增值税专用发票 | 01 | 可抵扣进项税 |
| 增值税普通发票 | 02 | 不可抵扣进项税 |
| 增值税电子专用发票 | 03 | 可抵扣进项税 |
| 增值税电子普通发票 | 04 | 不可抵扣进项税 |
| 通用机打发票 | 05 | 其他合法票据 |
| 收据 | 06 | 非正式票据 |
| 银行回单 | 07 | 银行凭证 |
| 其他 | 99 | 其他票据 |

#### 2.2 上传操作

1. 登录票据管理系统
2. 选择客户和会计期间
3. 点击"上传票据"按钮
4. 选择本地文件或拖拽上传
5. 系统自动识别票据数量和类型
6. 确认上传完成

#### 2.3 系统自动处理

上传后系统自动执行：
- 文件格式校验
- 文件大小检查（单文件≤10MB）
- 重复文件检测（MD5比对）
- 识别置信度评估

### Step 3: 票据识别

#### 3.1 OCR识别字段

| 字段名称 | 字段代码 | 说明 | 必填 |
|----------|----------|------|------|
| 发票类型 | invoice_type | 专用发票/普通发票/电子发票 | 是 |
| 发票代码 | invoice_code | 10位或12位 | 是 |
| 发票号码 | invoice_no | 8位 | 是 |
| 开票日期 | invoice_date | YYYY-MM-DD | 是 |
| 购买方名称 | buyer_name | 客户公司全称 | 是 |
| 购买方税号 | buyer_tax_no | 18位统一社会信用代码 | 是 |
| 销售方名称 | seller_name | 销货方公司全称 | 是 |
| 销售方税号 | seller_tax_no | 18位统一社会信用代码 | 是 |
| 货物或应税劳务名称 | goods_name | 商品/服务名称 | 是 |
| 规格型号 | spec | 规格（如有） | 否 |
| 单位 | unit | 计量单位 | 否 |
| 数量 | quantity | 数量 | 否 |
| 单价 | unit_price | 单价（不含税） | 否 |
| 金额 | amount | 不含税金额 | 是 |
| 税率 | tax_rate | 13%/9%/6%/3%/0% | 是 |
| 税额 | tax_amount | 税额 | 是 |
| 价税合计 | total_amount | 含税金额 | 是 |

#### 3.2 识别置信度处理

| 置信度范围 | 处理方式 | 系统行为 |
|------------|----------|----------|
| ≥95% | 自动填充 | 字段自动填充，绿色标注 |
| 80%-94% | 人工确认 | 字段黄色高亮，需人工确认 |
| <80% | 人工录入 | 字段红色高亮，需人工录入 |

### Step 5: 异常票据处理

#### 5.1 假发票识别

**识别规则**：
- 税务局数据库比对
- 发票代码+号码+金额+日期综合校验
- 销售方税务登记状态查询

**处理流程**：
```
发现疑似假票 → 标记SUSPICIOUS状态 → 人工核实 → 确认为假票→ 剔除并通知客户
                                                    → 确认为真票 → 解除标记
```

#### 5.2 过期票据处理

**识别规则**：
- 开票日期距今超过90天
- 特殊情况下可延长至180天（视税局要求）

**处理流程**：
```
发现逾期票据 → 标记OVERDUE状态 → 判断是否已抵扣
                                    → 已抵扣 → 需做进项税转出
                                    → 未抵扣 → 提示客户尽快处理
```

#### 5.3 金额异常处理

**识别规则**：
| 规则 | 条件 | 严重程度 |
|------|------|----------|
| 大额异常 | 单张金额 > 100,000 | 中 |
| 小额异常 | 单张金额 < 1（除特殊情况） | 中 |
| 负数异常 | 金额 < 0 | 高 |
| 零金额 | 金额 = 0 | 高 |

**处理流程**：
```
发现金额异常 → 标记AMOUNT_ABNORMAL → 人工核实业务真实性
                                              → 确认真实 → 保留并备注
                                              → 确认为误开 → 作废处理
```

#### 5.4 重复票据处理

**识别规则**：
- 同一发票代码+号码
- 同一销售方+开票日期+金额

**处理流程**：
```
发现重复票据 → 标记DUPLICATE → 比对入账情况
                                  → 未入账 → 保留一张，删除其余
                                  → 已入账 → 检查是否重复入账
                                               → 重复入账 → 红字冲销
                                               → 未重复 → 保留
```

### Step 6: 凭证生成

#### 6.1 一键生成凭证

**前置条件**：
- 所有票据状态为 CONFIRMED
- 无未处理的异常票据

**操作步骤**：
1. 选择会计期间
2. 点击"选中本月所有已确认票据"
3. 系统自动按票据类型分组
4. 自动匹配凭证模板
5. 生成凭证草稿
6. 用户审核确认

#### 6.2 分录规则

详见 `voucher-generation-rules.md`

#### 6.3 生成后状态更新

```json
{
  "operation": "generate_voucher",
  "invoice_ids": ["INV-20240115-001", "INV-20240115-002"],
  "voucher_id": "VCH-20240115-001",
  "status_change": {
    "before": "CONFIRMED",
    "after": "VOUCHERED"
  },
  "generated_at": "2024-01-31T18:00:00Z"
}
```

### Step 7: 归档

#### 7.1 月末结账归档

**归档时间**：每月末结账后

**归档操作**：
1. 检查当月所有票据是否已生成凭证
2. 执行结账操作
3. 批量更新状态为 ARCHIVED
4. 生成归档清单

#### 7.2 电子票据存档

**存档要求**：
- 按会计期间建立目录结构
- 文件命名包含归档日期
- 保留原始文件格式
- 备份至云存储

**目录结构**：
```
archive/
├── 2024/
│   ├── 01/
│   │   ├── 收入类/
│   │   ├── 成本类/
│   │   ├── 费用类/
│   │   ├── 银行类/
│   │   └── 特殊类/
│   └── 02/
│       └── ...
```

| 阶段 | 控制点 | 责任人 | 检查标准 |
|------|--------|--------|----------|
| 接收 | 票据完整性 | 客户/会计 | 按时提交、无遗漏 |
| 上传 | 文件合规性 | 会计 | 格式正确、命名规范 |

| 归类 | 分类正确性 | 会计 | 100%准确 |
| 异常 | 处理及时性 | 会计 | 24小时内处理 |
| 生成 | 借贷平衡 | 系统 | 100%平衡 |
